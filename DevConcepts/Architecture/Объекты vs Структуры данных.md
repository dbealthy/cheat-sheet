Использовать или не использовать методы в классах с публичными полями???

Лучше для классов с публичными полями использовать DTO (dataclass, pydantic) а для бизнес entity использовать объекты с методами, инкапсулирующие данные.

Например есть entity Product с публичными полями price: Money и name: str. Могут ли у него быть методы для валидации самого себя?? Могут быть ли другие методы??

Можно сделать синтаксическую валидацию через pydantic в DTO и сделать семантическую валидацию внутри entity


Entity инкапсулирует поля и предоставляет только нужные публичные методы и поля. Поэтому использование dataclass для entity считается антипаттерном

Entity является rich domain model и содержит не только данные но и методы, играющие ключевую роль для бизнеса
